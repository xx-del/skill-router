## 2. 是否需要本地模型？

### 分析结果

| 项目 | 结果 |
|------|------|
| **使用库** | bm25s |
| **算法类型** | BM25（词袋模型） |
| **是否需要向量模型** | ❌ 不需要 |
| **是否需要 LLM** | ❌ 不需要 |
| **是否需要本地模型** | ❌ 不需要 |

### 结论

**✅ 不需要本地模型**

**原因**：
- bm25s 是纯 Python 实现的 BM25 算法
- BM25 是词袋模型（Bag of Words），基于词频统计
- 不依赖神经网络、向量模型或 LLM
- 仅需 Numpy + Scipy（轻量级依赖）

### 对比

| 方案 | 需要模型 | 依赖 |
|------|----------|------|
| **skill-router（BM25）** | ❌ 不需要 | bm25s, Numpy, Scipy |
| **向量检索** | ✅ 需要 | sentence-transformers, 向量模型 |
| **LLM 路由** | ✅ 需要 | LLM API |

## 3. 是否使用了中文来初始化？

### 分析结果

| 项目 | 结果 |
|------|------|
| **测试用例语言** | 英文 |
| **中文测试用例** | 0/39 |
| **技能描述语言** | 中英文混合 |
| **查询语言支持** | 语言无关 |

### 结论

**⚠️ 未使用中文初始化**

**原因**：
- test_cases.json 全部使用英文查询
- SKILL.md 文件包含中英文混合描述
- BM25 算法是语言无关的

### BM25 对中文的支持

| 方面 | 支持程度 | 说明 |
|------|----------|------|
| **字符匹配** | ✅ 支持 | 中文字符可直接匹配 |
| **分词** | ⚠️ 基础 | 按空格/字符分割，无语义分词 |
| **语义理解** | ❌ 不支持 | 词袋模型，无法理解语义 |
| **推荐** | 💡 建议使用英文 | 英文分词效果更好 |

### 示例对比

```python
# 英文查询（效果好）
"frontend design interface" → frontend-design, sketch

# 中文查询（效果一般）
"前端设计界面" → 可能匹配不到（因为 SKILL.md 是英文描述）
```

### 改进建议

如需支持中文查询，可考虑：
1. **使用中文分词器**：jieba、pkuseg
2. **混合检索**：BM25 + 向量模型（支持语义）
3. **翻译查询**：中文查询翻译为英文后再匹配

```python

def scan_skills() -> List[Dict]:
    """动态扫描 Hermes 技能目录"""
    skills_dir = Path.home() / '.hermes' / 'skills'
    
    for skill_md in skills_dir.rglob('SKILL.md'):
        # 解析 SKILL.md
        content = skill_md.read_text(encoding='utf-8')
        frontmatter = parse_frontmatter(content)
        # ...
```

### 对比

| 方案 | 方式 | 优点 | 缺点 |
